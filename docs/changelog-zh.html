v1.0.1 2026年1月8日

🔧 更改

  • 在manifest.json中添加了clipboardWrite权限，以明确剪贴板访问权限
  • 提高了商店审核流程的权限清晰度

📋 技术细节

  • 扩展使用剪贴板API自动将AI处理的文本复制到剪贴板
  • 以前仅依赖scripting权限来注入剪贴板访问代码
  • 现在明确声明clipboardWrite权限以提高透明度

✨ 为什么要做这个更改？

  • 清晰度：在权限列表中明确剪贴板使用
  • 合规性：确保完全符合商店要求
  • 面向未来：为service worker中潜在的直接剪贴板API访问做准备

注意：无功能更改。用户体验保持不变。

v1.0.0 2026年1月5日

🎉 首次发布

✨ 功能特性

  • 通过右键上下文菜单进行AI驱动的文本增强
  • 5个内置操作：润色和改进、修复语法、使其专业、简化语言、总结
  • 可自定义操作，支持用户定义的提示词
  • 支持任何OpenAI兼容的API（OpenAI、Azure、本地LLM）
  • 自动剪贴板集成
  • 带徽章通知的实时视觉反馈
  • Chrome系统通知用于状态更新
  • 国际化支持（i18n就绪）
  • 隐私优先设计 - 所有数据本地存储

🔐 权限

  • contextMenus - 右键菜单集成
  • storage - 本地保存用户设置
  • notifications - 显示状态更新
  • activeTab - 访问选定的文本
  • scripting - 执行剪贴板复制脚本
  • host_permissions - 连接到用户配置的AI端点
